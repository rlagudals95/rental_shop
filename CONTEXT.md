# CONTEXT.md

이 문서는 rental_shop 프론트엔드의 제품/도메인 맥락을 담습니다.

## 1) Product Goal

렌탈샵 서비스의 핵심 사용자 흐름(상품 탐색 → 대여 가능 여부 확인 → 예약/문의)을 빠르고 명확하게 제공하는 프론트엔드를 구축한다.

## 2) Primary Users

- 일반 고객: 대여 가능한 상품을 빠르게 찾고 예약하고 싶음
- 운영자(향후): 재고/예약 상태를 관리하고 싶음

## 3) Core User Flow (현재 기준)

1. 랜딩/홈 진입
2. 카테고리 또는 검색으로 상품 탐색
3. 상품 상세에서 가격/대여 기간/재고 상태 확인
4. 예약 또는 문의 액션
5. 완료/확인 피드백

## 4) UX Principles

- 모바일 우선(터치/가독성 우선)
- 빠른 인지(핵심 정보: 가격, 기간, 재고를 상단에)
- 불필요한 클릭 최소화
- 상태 명확성(로딩/에러/빈 상태 UI 필수)

## 5) Engineering Priorities

1. 안정성: 타입 안정성 + 린트/포맷 일관성
2. 유지보수성: 도메인 기준 폴더 구조, 작은 컴포넌트
3. 확장성: 향후 예약/결제/관리자 기능 확장 고려
4. 성능: 불필요한 클라이언트 컴포넌트 최소화

## 6) Out of Scope (지금은 보류)

- 결제 연동
- 관리자 대시보드 전체
- 고급 추천 알고리즘

## 7) Terminology

- 재고(Stock): 실제 대여 가능한 수량
- 대여 기간(Rental Duration): 시작~종료 날짜/시간
- 예약(Booking): 사용자의 대여 요청 확정 데이터
- 블랙아웃(Blackout): 대여 불가 시간대/날짜

## 8) Notes

- 큰 도메인 변경/정책 결정은 `DECISIONS.md`에 기록한다.
- 현재 컨텍스트는 초기 버전이며, 기능 구현 시 지속 업데이트한다.
