# PRD - 렌트에이전트 (가칭)

## 1. Summary

이 문서는 정수기 렌탈을 고민하는 사용자를 위한 "계약 해석형 추천 에이전트"의 MVP 요구사항을 정의한다. 서비스는 상품을 많이 보여주는 대신, 사용자 상황에 맞는 3개 상품만 추천하고 계약 리스크를 쉽게 설명한다. 목표는 비교 피로를 줄이고 상담 전환을 높이는 것이다.

## 3. Background

### Context

국내 렌탈 시장에는 강한 사업자가 이미 많다. 신규 팀이 설치망이나 가격만으로 승부하기는 어렵다. 반면 사용자는 계약 구조를 이해하기 어렵고, 비교에 많은 시간을 쓴다.

### Why now

정수기 렌탈은 여전히 큰 카테고리다. 하지만 계약해지, 위약금, 할인 종료 후 금액 같은 내용에서 불만이 반복된다. 이 문제는 "더 많은 상품"이 아니라 "더 쉬운 해석"으로 풀 수 있다.

### What changed

구조화된 상품 데이터와 간단한 룰 엔진, LLM 기반 설명 레이어를 결합하면 소규모 팀도 빠르게 MVP를 만들 수 있다. 즉, AI가 판단을 대신하는 것이 아니라 판단 근거를 쉽게 설명하는 UX를 빠르게 출시할 수 있다.

### Evidence (to keep validating)

- 2022~2025년 1분기 정수기 렌탈 피해구제 1,462건, 2024년 536건 보도(한국소비자원 발표 인용 기사 다수)
- 계약 관련 불만 비중 56.3%, 그중 계약해지/위약금 61.1% 보도
- 최근에는 "의무사용기간 종료 후에도 비용 발생" 고지 강화 이슈가 다시 부각됨

## 4. Objective

### Objective

사용자가 정수기 렌탈 계약을 이해한 상태에서, 본인에게 맞는 3개 상품만 빠르게 결정하도록 돕는다.

### Why it matters

- 고객: 월요금만 보고 가입하는 실수를 줄인다.
- 회사: 상담 전환 품질을 높이고, 제휴 수익 실험이 가능해진다.
- 전략: 렌탈몰 경쟁이 아니라 "계약 해석 + 개인화 추천 + 상담 전환"에 포지셔닝한다.

### Key Results (SMART)

- KR1. 온보딩 시작 사용자 중 완료율을 8주 내 **65% 이상** 달성
- KR2. 추천 결과 조회 사용자 중 상담 CTA 클릭률을 8주 내 **20% 이상** 달성
- KR3. 추천 결과 조회 사용자 중 상담 신청률을 8주 내 **10% 이상** 달성
- KR4. 사용자 설문에서 "계약 이해가 쉬웠다" 응답을 8주 내 **70% 이상** 달성
- KR5. 온보딩 시작부터 추천 결과 도달까지 소요시간 P75를 **60초 이내** 유지

## 5. Market Segment(s)

시장은 인구통계보다 "해결하려는 일"로 정의한다.

### Segment A: 빠르게 결정해야 하는 사용자

- Job: 짧은 시간 안에 후보를 3개 이하로 줄이고 싶다.
- Situation: 바쁜 직장인, 정보 탐색 시간 부족.
- Constraint: 긴 설명보다 바로 비교 가능한 요약이 필요.

### Segment B: 해지 리스크가 걱정되는 사용자

- Job: 나중에 손해 보는 계약을 피하고 싶다.
- Situation: 전월세, 이사 가능성 높음.
- Constraint: 의무사용기간/계약기간/해지비용을 쉽게 봐야 함.

### Segment C: 관리 방식까지 따져야 하는 사용자

- Job: 방문관리 vs 셀프관리 중 내 생활에 맞는 방식을 고르고 싶다.
- Situation: 가격 민감 + 유지관리 부담 걱정.
- Constraint: 월요금만이 아니라 관리 방식의 trade-off 설명 필요.

### Initial Scope Constraints

- 카테고리는 정수기부터 시작
- 1~2인 가구 중심 시나리오 우선
- 수도권 설치/상담 가능 상품 우선 수집

## 6. Value Proposition(s)

### Customer jobs / pains / gains

- Job: "나한테 맞는 렌탈 상품을 빨리 고른다"
- Pain: 정보가 많고 계약 문구가 어려워서 결정이 늦다
- Gain: 1분 내 요약 + 3개 추천 + 리스크 안내로 결정 부담이 줄어든다

### What we solve better

기존 렌탈몰은 상품 노출과 가격 혜택 중심이다. 본 서비스는 계약 해석과 개인 상황 맞춤 설명에 집중한다.

### Value Curve (MVP 기준)

| 비교 축          | 일반 렌탈몰 | 렌트에이전트         |
| ---------------- | ----------- | -------------------- |
| 상품 개수 노출   | 높음        | 낮음(의도적으로 3개) |
| 계약 구조 가독성 | 낮음        | 높음                 |
| 개인화 추천 설명 | 중간/낮음   | 높음                 |
| 해지 리스크 안내 | 낮음        | 높음                 |
| 상담 전 결정도   | 중간        | 높음                 |

## 7. Solution

### 7.1 UX / User Flow

1. 랜딩: "월요금만 보면 손해" 메시지 + 1분 추천 CTA
2. 대화형 온보딩: 7~9문항으로 상황 수집
3. AI 요약: 사용자 조건을 한 문단으로 정리
4. 추천 결과: 상위 3개 + 비교표 + 해지 주의 포인트
5. 상담 전환: 카카오/폼/전화 예약 중 선택

### 7.2 Key Features

#### A. 대화형 온보딩

- 입력: 가구 수, 거주 형태, 이사 가능성, 예산, 기능 니즈, 관리 선호, 공간 제약, 최대 걱정
- 요구사항: 진행률 표시, 이탈 후 재진입, 모바일 터치 최적화

#### B. 추천 엔진 (초기 룰 기반)

- 핵심 규칙 예시:
  - `moving_within_24m=true`면 장기계약 감점
  - `budget_range=low`면 저요금 상품 가점
  - `care_preference=self`면 셀프관리형 가점
  - `ice=required`면 얼음 미지원 상품 제외
- 출력: 상위 3개 + 추천 이유 + 제외 이유

#### C. 계약 요약/리스크 설명 레이어

- 구조화된 상품 필드만 사용해 설명 생성
- 금지: 근거 없는 추정, 허위 할인 정보, 불확실 비용 확정 표현

#### D. 결과 페이지

- 카드 필수 항목: 월 납부액, 할인 종료 후 금액, 의무사용기간, 계약기간, 총 예상 납부액, 관리 방식, 해지 주의 포인트
- CTA: "설치 가능 조건 확인", "비교 결과 전달 후 상담"

#### E. 운영 관리자 (초기형)

- 상품 등록/수정/숨김
- 계약 조건 수정
- 추천 규칙 수정
- 상담 상태 업데이트 및 리드 export

### 7.3 Technology (MVP)

- Frontend: Next.js App Router + TypeScript
- Data: 정수기 상품 구조화 테이블(관리형/셀프형, 계약 필드 포함)
- Logic: 서버 측 룰 엔진 + 점수 계산
- AI: 설명 생성 전용 레이어(추천 판단은 룰 엔진이 담당)
- Analytics: 온보딩/결과/CTA 이벤트 로깅

### 7.4 Assumptions to Validate

- A1. 사용자는 "최저가"보다 "계약 이해"에 더 높은 가치를 둔다.
- A2. 추천 결과를 3개로 제한하면 상담 전환 품질이 올라간다.
- A3. 계약 리스크 설명이 있으면 상담 이탈이 줄어든다.
- A4. 정수기 단일 카테고리만으로 초기 PMF 신호를 확인할 수 있다.

## 8. Release

### Scope and timeline (relative)

- **Phase 0 (1주)**: 인터뷰 10명, 질문지 확정, SKU 20~30개 정리
- **Phase 1 (2주)**: 랜딩 + 온보딩 + 추천 3개 결과 화면 출시
- **Phase 2 (1주)**: 상담 전환, 운영 관리자 최소 기능, 이벤트 대시보드

### v1 포함

- 정수기 단일 카테고리
- 룰 기반 추천
- AI 요약/설명
- 상담 전환(카카오/폼)
- 운영자 상품/리드 관리

### v1 제외

- 결제, 설치 스케줄링, 자산 보유 렌탈
- 다카테고리 동시 운영
- 실시간 입점 API
- 모바일 앱

### Post-v1

- 공기청정기/비데 확장
- 규칙 자동 튜닝(A/B 테스트)
- 상담 대화 품질 점수화
