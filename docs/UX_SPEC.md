# UX Spec - Rental Decision Flow (v1)

## 1. 설계 목표

핵심 목표:

- 사용자가 60초 안에 추천 결과까지 도달한다.
- 월요금 중심 비교를 "계약 해석 중심 비교"로 바꾼다.
- 결과 페이지에서 상담 전환을 자연스럽게 유도한다.

핵심 원칙:

- 모바일 우선
- 한 화면 한 행동
- 숫자와 문장을 함께 보여준다
- 로딩/에러/빈 상태를 반드시 명시한다

## 2. 정보 구조 (IA)

1. `/` 랜딩
2. `/funnel/apply` 온보딩 + 신청 입력
3. `/funnel/complete` 완료
4. `/product/[slug]` 상품 상세
5. `/admin/mock-leads` 운영 리드 확인

MVP 의사결정 흐름:
`랜딩 -> 온보딩 -> 추천 결과(3개) -> 상담 전환 -> 완료`

## 3. 사용자 여정

## 3.1 Step A. 랜딩

사용자 질문: "어디서 시작해야 하지?"

해결 UI:

- 헤드라인: "월요금만 보면 손해"
- 보조 문구: "의무사용기간/계약기간 분리 안내"
- CTA 1: "1분 추천 시작"
- CTA 2: "상품 먼저 보기"

성공 기준:

- 랜딩 대비 온보딩 시작률 35%+

## 3.2 Step B. 온보딩

사용자 질문: "내 상황을 빨리 입력할 수 있나?"

해결 UI:

- 질문 7~9개
- 대부분 칩 선택형
- 진행률 표시(예: 3/8)
- 이전/다음 버튼 고정

성공 기준:

- 시작 대비 완료율 65%+
- 이탈 상위 질문 파악 가능

## 3.3 Step C. 추천 결과

사용자 질문: "왜 이 3개가 나왔지?"

해결 UI:

- 사용자 조건 요약 1문단
- 추천 카드 3개 고정
- 각 카드에 추천 이유/리스크 노출
- 하단 비교표에서 핵심 숫자 정렬

성공 기준:

- 결과 노출 대비 CTA 클릭률 20%+

## 3.4 Step D. 상담 전환

사용자 질문: "상담 시 다시 처음부터 설명해야 하나?"

해결 UI:

- "비교 결과 전달 후 상담" CTA
- 상담 폼 최소 입력(이름, 연락처, 희망 채널)
- 추천 결과 자동 첨부 안내

성공 기준:

- 결과 페이지 대비 상담 신청률 10%+

## 4. 화면별 상세 명세

## 4.1 Screen 01 - Landing (`/`)

필수 섹션:

- Hero
- 가치 제안 3개
- 추천 결과 샘플
- 신뢰 근거(계약 이슈 데이터)
- 1차 CTA/2차 CTA

컴포넌트:

- `HeroBanner`
- `ValueCard`
- `SampleResultCard`
- `PrimaryCTA`

상태:

- 기본 상태만 존재 (정적 페이지)

## 4.2 Screen 02 - Onboarding (`/funnel/apply` 1단계)

입력 항목:

- 가구 수
- 거주 형태
- 24개월 내 이사 가능성
- 월 예산
- 필수 기능(냉/온/정/얼음)
- 관리 선호(방문/셀프)
- 공간 제약
- 가장 큰 걱정(가격/관리/해지)

인터랙션:

- 선택 후 자동 다음 질문 이동(모바일 속도 최적화)
- 직접 입력은 마지막 1문항만 허용
- 뒤로가기 시 기존 답 유지

오류 처리:

- 필수값 누락 시 하단 인라인 에러 노출
- 네트워크 에러 시 재시도 버튼

## 4.3 Screen 03 - Result (신규 화면, `/funnel/result` 권장)

상단:

- "당신에게 중요한 기준" 요약

중단:

- 추천 카드 3개
- 카드 필수 필드:
  - 월 납부액
  - 할인 종료 후 금액
  - 의무사용기간
  - 계약기간
  - 총예상납부액
  - 관리 방식
  - 해지 주의 포인트

하단:

- 비교 테이블
- CTA: "설치 가능 조건 확인" / "비교 결과 전달 후 상담"

빈 상태:

- 추천 결과가 없으면 조건 완화 가이드 제공

## 4.4 Screen 04 - Apply Step2 (`/funnel/apply?step=2`)

필수 입력:

- 설치 지역
- 희망 연락 시간
- 메모

UX 포인트:

- 단계 표시(1/2, 2/2)
- 이전 단계 복귀 가능
- 제출 완료 시 `/funnel/complete` 이동

## 4.5 Screen 05 - Complete (`/funnel/complete`)

필수 요소:

- 접수 완료 메시지
- 접수 정보 요약
- 예상 연락 시간
- 재탐색 버튼, 홈 버튼

## 4.6 Screen 06 - Admin (`/admin/mock-leads`)

필수 요소:

- 리드 목록
- 상태 필터
- 상세 메모
- 최신순 정렬

## 5. 컴포넌트 설계 가이드

공통 컴포넌트 제안:

- `FunnelProgress`
- `QuestionCard`
- `OptionChipGroup`
- `RecommendationCard`
- `RiskNote`
- `ComparisonTable`
- `LeadCTA`

컴포넌트 공통 원칙:

- 터치 영역 최소 44px
- 제목/본문 대비 명확
- 버튼 문구는 행동 중심 동사 사용

## 6. 카피라이팅 가이드

좋은 예:

- "의무사용기간과 계약기간을 나눠 보여드려요"
- "이사 가능성이 있으면 이 조건을 먼저 보세요"

피해야 할 문구:

- "무조건 이게 최고"
- "위약금 걱정 없음"
- "AI가 전부 해결"

## 7. 상태 설계

필수 상태 정의:

- Loading: 스켈레톤 사용
- Empty: 조건 완화 유도 문구
- Error: 원인 + 재시도
- Success: 다음 행동 버튼 제공

오류 메시지 규칙:

- 사용자 책임처럼 보이지 않게 표현
- 해결 방법을 한 줄로 함께 안내

## 8. 접근성 기준

- 명도 대비 WCAG AA 이상
- 키보드 포커스 표시 유지
- 스크린리더용 레이블 명시
- 오류 문구를 입력 필드와 연결

## 9. 이벤트 측정 설계

퍼널 이벤트:

- `landing_view`
- `onboarding_start`
- `onboarding_question_answered`
- `onboarding_complete`
- `result_view`
- `result_cta_click`
- `lead_submit`

속성:

- `session_id`
- `question_key`
- `selected_option`
- `recommended_product_ids`
- `channel`

## 10. 반응형 규칙

브레이크포인트:

- Mobile: <= 767px
- Tablet: 768~1023px
- Desktop: >= 1024px

레이아웃:

- Mobile: 단일 컬럼, 하단 고정 CTA
- Tablet: 단일 컬럼 + 여백 확장
- Desktop: 2컬럼(요약/카드 + 비교표)

## 11. 디자인 토큰 제안

- Primary: `#111111`
- Accent: `#F36F00`
- Surface: `#F6F7F9`
- Border: `rgba(0,0,0,0.10)`
- Danger note bg: `#FFF3E8`

타이포:

- Title: 24/32, 700+
- Body: 14/22, 400~500
- Caption: 12/18, 400

## 12. 구현 우선순위

Sprint 1:

- 랜딩 메시지 개편
- 온보딩 질문 구조 확정
- 결과 화면 라우트 신설

Sprint 2:

- 추천 카드/비교표 완성
- 상담 전환 폼 연결
- 이벤트 로깅

Sprint 3:

- 관리자 화면 고도화
- 카피 A/B 테스트
- 접근성 개선

## 13. 승인 기준 (UX)

출시 전 체크리스트:

- 주요 경로 60초 내 완료 가능
- 모바일에서 모든 버튼 44px 이상
- 필수 상태(로딩/에러/빈 상태) 모두 구현
- 결과 화면에서 추천 이유와 리스크를 동시에 확인 가능
- 상담 전환 시 추천 요약이 자동 전달됨
