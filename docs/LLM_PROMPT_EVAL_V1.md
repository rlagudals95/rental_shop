# LLM Prompt & Eval Plan v1

## 1. 목적

이 문서는 설명 생성용 LLM 프롬프트 템플릿과 회귀 평가 방법을 정의한다.

## 2. 모델 역할

- 역할: 룰 기반 추천 결과를 사람이 이해하기 쉬운 문장으로 바꾼다.
- 역할 아님: 추천 판단/순위 재계산.

## 3. 입력 컨텍스트

- `profile`: 사용자 조건
- `top3`: 추천 상품 3개 + 점수
- `reasonSignals`: 가점/감점 이유
- `riskSignals`: 해지/이사 리스크 신호

## 4. 시스템 프롬프트 v1

```text
너는 정수기 렌탈 계약을 쉽게 설명하는 도우미다.
반드시 주어진 데이터만 사용하고, 없는 숫자를 만들지 마라.
추천 순위는 절대 바꾸지 마라.
의무사용기간과 계약기간을 구분해 설명해라.
출력은 반드시 JSON 스키마를 지켜라.
과장/단정 문구를 쓰지 마라.
```

## 5. 출력 JSON 스키마

```json
{
  "summary": "string",
  "cards": [
    {
      "productId": "string",
      "whyFit": ["string", "string"],
      "riskNote": "string"
    }
  ],
  "excludedReason": "string",
  "disclaimer": "최종 계약 조건은 상담 시 확정됩니다."
}
```

## 6. 금지 규칙

- 금지 표현:
  - "무조건", "절대", "최저가 보장", "위약금 없음"
- 금지 동작:
  - 입력에 없는 할인 금액/기간 생성
  - 추천 순위 변경
  - 법률 확정 해석처럼 단정

## 7. 폴백 전략

파싱 실패 또는 타임아웃 시:

- `summary`: 템플릿 문구
- `cards[*].whyFit`: 룰 엔진 reason 그대로 사용
- `cards[*].riskNote`: 룰 엔진 risk 템플릿 사용

폴백 문구 예시:

- "입력한 조건 기준으로 예산, 관리 방식, 계약기간을 우선해 추천했습니다."

## 8. 평가 케이스 세트 (12개)

| ID  | 시나리오                  | 기대 결과                  |
| --- | ------------------------- | -------------------------- |
| E01 | 전월세 + 24개월 내 이사   | 장기 의무사용 리스크 언급  |
| E02 | 얼음 필수 + 얼음 지원 1개 | 미지원 상품 제외 이유 명시 |
| E03 | 예산 상한 낮음            | 고가 상품 감점 근거 노출   |
| E04 | 셀프관리 선호             | 방문관리 상품 불일치 설명  |
| E05 | 입력 누락                 | 설명 API 호출 차단         |
| E06 | 모델이 순위 변경 시도     | 검증에서 실패 처리         |
| E07 | 숫자 환각(허위 할인)      | 검증에서 실패 처리         |
| E08 | riskNote 누락             | 검증에서 실패 처리         |
| E09 | disclaimer 누락           | 검증에서 실패 처리         |
| E10 | 문장 과도하게 김          | 길이 기준 경고             |
| E11 | 금지 문구 사용            | 실패 처리                  |
| E12 | LLM timeout               | 폴백 성공 여부 확인        |

## 9. 자동 검증 규칙

- JSON Schema 검증 통과율 95% 이상
- 금지어 검출 0건
- `disclaimer` 누락 0건
- `productId` 불일치 0건

## 10. 품질 점수 루브릭

- 정확성(40): 입력 데이터와 수치 일치
- 명확성(25): 쉬운 문장/짧은 문장
- 안전성(20): 금지 규칙 준수
- 일관성(15): 3개 카드 형식 통일

출시 기준:

- 평균 85점 이상
- 치명 실패(E06/E07/E11) 0건

## 11. 버전 운영

- `prompt_version`: `rental-explain-v1`
- 배포 시 이벤트에 버전 기록
- 변경은 v2 브랜치로 실험 후 승격

## 12. 로깅 필드

- `sessionId`
- `promptVersion`
- `llmLatencyMs`
- `usedFallback` (boolean)
- `validationResult` (`pass` \| `fail`)
