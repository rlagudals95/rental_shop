# Wireframe Spec - Funnel Result (`/funnel/result`)

## 1. 목적

이 문서는 추천 결과 화면을 바로 구현하기 위한 구조/상태/컴포넌트 명세다.

## 2. 라우트 계약

- 경로: `/funnel/result`
- 입력: `sessionId` (query 또는 server session)
- 선행 조건: 온보딩 완료 상태(`PROFILE_READY` 이상)

미충족 시:

- 온보딩으로 리다이렉트: `/funnel/apply?step=1`

## 3. 모바일 와이어프레임

```text
┌──────────────────────────────┐
│ [헤더] 추천 결과             │
│ 당신에게 중요한 기준 3가지   │
├──────────────────────────────┤
│ [요약 카드]                  │
│ 1인 가구 / 전월세 / 이사가능 │
│ 해지 부담 낮은 조건 우선     │
├──────────────────────────────┤
│ [추천 카드 #1]               │
│ 브랜드 + 모델                │
│ 월 27,900 (할인후 31,900)    │
│ 의무 36 / 계약 48            │
│ 추천 이유 2줄                │
│ 해지 주의 1줄                │
│ [이 상품으로 상담]           │
├──────────────────────────────┤
│ [추천 카드 #2] ...           │
├──────────────────────────────┤
│ [추천 카드 #3] ...           │
├──────────────────────────────┤
│ [비교표 토글] 펼치기         │
│ 월요금 / 계약기간 / 관리방식 │
├──────────────────────────────┤
│ [고정 하단 CTA]              │
│ 비교 결과 전달 후 상담하기   │
└──────────────────────────────┘
```

## 4. 데스크탑 와이어프레임

```text
┌───────────────────────────────────────────────────────────┐
│ 헤더                                                     │
├───────────────────────────────┬───────────────────────────┤
│ 왼쪽: 사용자 요약 + 카드 3개   │ 오른쪽: 고정 비교표       │
│                               │ 월요금/의무/계약/관리     │
│                               │ + 상담 CTA                │
└───────────────────────────────┴───────────────────────────┘
```

## 5. 컴포넌트 트리

- `ResultPage`
- `ProfileSummaryCard`
- `RecommendationCardList`
- `RecommendationCard`
- `RiskNote`
- `ComparisonTable`
- `LeadStickyCTA`
- `ResultErrorState`
- `ResultEmptyState`

## 6. 카드 필수 필드

- 상품명
- 월 납부액(`promoFee`, `postPromoFee`)
- 의무사용기간
- 전체 계약기간
- 관리 방식
- 추천 이유(2개)
- 해지 주의 포인트(1개)
- CTA 버튼

## 7. 인터랙션 규칙

- 카드 클릭 시 상세 펼침(추천 이유/제외 이유)
- 비교표는 기본 접힘(모바일), 펼침 상태 기억
- 선택 CTA 클릭 시 `selectedProductId` 저장 후 상담 화면 이동

## 8. 상태 명세

## 8.1 Loading

- 스켈레톤 3카드 + 요약 카드 1개
- 1.5초 이상이면 "추천 계산 중" 텍스트 표시

## 8.2 Empty

- 문구: "조건에 맞는 상품이 없습니다"
- 액션: "조건 다시 설정" 버튼(온보딩 복귀)

## 8.3 Error

- 문구: "결과를 불러오지 못했어요"
- 액션: "다시 시도" / "온보딩으로 돌아가기"

## 8.4 Partial (설명 실패)

- 카드 노출은 유지
- 설명 대신 템플릿 문구 사용
- 배너: "설명 생성이 지연되어 기본 설명으로 안내 중"

## 9. 이벤트 설계

- `result_view`
- `result_card_expand`
- `result_compare_open`
- `result_product_select`
- `result_cta_lead_click`

필수 속성:

- `sessionId`
- `recommendedProductIds`
- `selectedProductId`
- `renderMode` (`normal` \| `fallback`)

## 10. 접근성

- 카드 CTA `aria-label`에 상품명 포함
- 비교표는 `caption` 포함
- 키보드 포커스 순서: 요약 -> 카드1 -> 카드2 -> 카드3 -> 비교표 -> CTA
