# PRD - 렌트에이전트 (v2, 전략 강화)

## 1. Summary

이 문서는 정수기 렌탈을 고르는 고객을 위한 "계약 해석 + 3개 추천" 서비스의 MVP 요구사항이다. 우리는 렌탈몰처럼 상품을 많이 보여주지 않는다. 대신 계약 리스크를 쉽게 설명하고, 상담 전에 결정을 거의 끝내게 만든다.

## 3. Background

### 3.1 Context

국내 렌탈 시장은 이미 큰 사업자가 강하게 자리 잡고 있다. 신규 팀이 설치망, 브랜드 신뢰, 가격만으로 이기기는 어렵다. 그래서 우리는 "상품 수"가 아니라 "의사결정 품질"에 집중한다.

### 3.2 Why now (데이터 근거)

공정거래위원회 2025년 7월 24일 보도자료에 따르면:

- 2022년 1월~2025년 3월 정수기 렌탈 피해구제 신청: **1,462건**
- 2024년 신청 건수: **536건**(연도 기준 증가)
- 불만 유형 1위: 계약 관련 **56.3%**
- 계약 불만 중 1위: 계약해지·위약금 **61.1%**
- 품목 비중 상위: 정수기 **43.2%**, 공기청정기 **12.4%**, 비데 **10.4%**

또한 같은 보도자료는 "의무사용기간이 끝나도 계약기간이 남아 있으면 이용료가 발생할 수 있음"을 명시한다. 즉, 고객은 "월요금"만 보면 오판하기 쉽다.

### 3.3 What became possible

이제 소규모 팀도 아래 조합으로 빠르게 MVP를 만들 수 있다.

- 구조화된 계약 데이터
- 룰 기반 추천 엔진
- AI 설명 레이어(판단 근거를 쉽게 번역)

핵심은 "AI가 판단"이 아니다. "AI가 이해를 돕는 것"이다.

## 4. Objective

### 4.1 Product Objective

사용자가 자기 상황에 맞는 정수기 렌탈 3개를 빠르게 고르고, 해지/위약금 리스크를 이해한 상태로 상담까지 연결되게 한다.

### 4.2 Strategic Focus (뾰족한 포지션)

우리는 초기 8주 동안 아래 1개 문제만 푼다.

- **정수기 × 전월세/이사 가능성 있음 × 계약해지 불안이 큰 사용자**

우리는 아래 경쟁은 하지 않는다.

- 최저가 경쟁
- 사은품 경쟁
- 카테고리 확장 경쟁

### 4.3 Why this matters

- 고객 가치: 후회 가능성이 큰 계약을 미리 걸러준다.
- 회사 가치: 상담 전환의 "양"보다 "질"을 올린다.
- 전략 정합성: 운영망 없이도 데이터/설명 UX로 차별화 가능하다.

### 4.4 Key Results (SMART)

- KR1. 온보딩 완료율: 출시 후 8주 내 **65%+**
- KR2. 결과 페이지 도달 사용자 중 상담 CTA 클릭률: 8주 내 **20%+**
- KR3. 결과 페이지 도달 사용자 중 상담 신청률: 8주 내 **10%+**
- KR4. 사용자 설문("계약 이해가 쉬웠다") 긍정 응답률: 8주 내 **70%+**
- KR5. 온보딩 시작~추천 결과 도달 시간 P75: **60초 이내**

## 5. Market Segment(s)

시장 정의는 연령보다 "해결할 일(Job)"로 한다.

### 5.1 Beachhead Segment (초기 집중 세그먼트)

- Job: "긴 비교 없이, 손해 가능성이 낮은 상품을 고르고 싶다"
- 상황: 바쁜 직장인, 1~2인 가구, 전월세, 2년 내 이사 가능성 있음
- 제약: 긴 약관을 읽을 시간과 에너지가 부족함

### 5.2 Secondary Segment (확장 후보)

- 공기청정기 렌탈 사용자 (동일한 계약/관리 구조)
- 비데 렌탈 사용자 (계약/관리 방식 비교 니즈 유사)

### 5.3 Out of scope (MVP)

- 기업/법인 렌탈
- 고관여 대형가전(안마의자, 매트리스)
- 자체 결제/설치 스케줄링

## 6. Value Proposition(s)

### 6.1 Customer jobs / pains / gains

- Job: 내 상황에 맞는 상품만 빠르게 고른다.
- Pain: 월요금 중심 비교로 계약 리스크를 놓친다.
- Gain: 총비용, 계약기간, 해지 주의점을 한 번에 본다.

### 6.2 Core Promise

"렌탈 계약을 대신 읽어주고, 당신에게 맞는 3개만 보여준다."

### 6.3 Value Curve (초기)

| 축                    | 일반 렌탈몰 | 렌트에이전트    |
| --------------------- | ----------- | --------------- |
| 노출 상품 수          | 높음        | 낮음(3개 고정)  |
| 계약 가독성           | 낮음        | 높음            |
| 이사/해지 리스크 안내 | 낮음        | 높음            |
| 상담 전 결정도        | 중간        | 높음            |
| 운영 복잡도           | 높음        | 낮음(집중 운영) |

### 6.4 Defensible edge (초기 방어력)

- 계약 필드 표준화 DB (월요금/할인종료요금/의무사용/계약기간/해지조건)
- "이사 가능성" 반영 리스크 스코어
- 상담 전 전달용 요약 리포트(상담 품질 표준화)

## 7. Solution

### 7.1 UX / User Flow

1. 랜딩: "월요금만 보면 손해" 메시지 + 1분 시작 CTA
2. 온보딩(7~9문항): 가구, 거주형태, 이사 가능성, 예산, 관리 선호, 우려사항 입력
3. AI 요약: 사용자 조건과 우선순위를 쉬운 문장으로 정리
4. 추천 결과: 상위 3개 + 공통 비교표 + 해지 주의점
5. 상담 전환: 카카오/폼으로 연결, 비교 결과 자동 전달

### 7.2 Key Features (MVP)

#### Feature A. 대화형 온보딩

- 요구사항: 진행률 표시, 중단 후 재개, 모바일 최적화
- 목표: 60초 내 완료 가능

#### Feature B. 룰 기반 추천 엔진

- 예시 룰:
  - `moving_within_24m=true` → 장기 의무사용 감점
  - `care_preference=self` → 셀프관리 가점
  - `ice=required` → 얼음 기능 없는 제품 제외
  - `budget_range=low` → 월요금/총요금 가중치 상향
- 출력: 추천 사유 + 제외 사유 + 리스크 문장

#### Feature C. 계약 해석 카드

- 카드 필수 노출:
  - 월요금
  - 할인 기간 종료 후 요금
  - 의무사용기간
  - 전체 계약기간
  - 총예상납부액
  - 중도해지 시 주의 포인트

#### Feature D. 상담 전환 + 운영자 화면

- 리드 생성, 상담 상태 관리, 인터뷰 메모
- 운영자가 계약 필드와 규칙을 즉시 수정 가능

### 7.3 Technology

- Next.js + TypeScript
- 상품/계약 데이터 구조화 테이블
- 서버 추천 로직(룰 + 점수)
- AI는 설명 생성 전용(판단은 규칙이 담당)
- 이벤트 로깅: 시작/완료/결과조회/CTA/상담신청

### 7.4 Assumptions (검증 대상)

- A1. 사용자는 최저가보다 "해석된 계약 정보"에 더 반응한다.
- A2. 3개 추천 제한이 선택 피로를 줄여 상담 전환을 올린다.
- A3. 이사 가능성 기반 리스크 안내가 신뢰를 높인다.
- A4. 정수기 단일 카테고리만으로도 유의미한 전환 학습이 가능하다.

## 8. Release

### 8.1 Timeline (상대 일정)

- Phase 0 (1주): 인터뷰 10명, 질문지/필드 확정, SKU 20~30개 수집
- Phase 1 (2주): 랜딩 + 온보딩 + 결과 3개 + 계약 해석 카드
- Phase 2 (1주): 상담 전환, 운영자 화면, 퍼널 대시보드

### 8.2 v1 포함

- 정수기 단일 카테고리
- 룰 기반 추천 + AI 설명
- 상담 연결(카카오/폼)
- 운영자 데이터 수정 기능

### 8.3 v1 제외

- 결제, 설치 스케줄링, 자체 렌탈 자산
- 다카테고리 동시 운영
- 실시간 입점 API
- 모바일 앱

### 8.4 Go / No-Go Gate (8주)

- Go: KR2/KR3 동시 달성 + "계약 이해" 긍정 70% 이상
- No-Go: 상담 신청률 5% 미만이면 포지셔닝/세그먼트 재설계
- Fix-first: 온보딩 완료율 50% 미만이면 질문 수/순서 먼저 축소
